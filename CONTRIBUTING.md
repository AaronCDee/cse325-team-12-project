# Contributing

## Branch Workflow

- Keep `main` deployable and use it only for reviewed changes.
- Create feature branches from `main` using the format `wk<week>-<name>`.
- For this sprint, use `wk4-boluwatife`.
- Open a pull request back into `main` when your work is ready for review.

## Working Agreement

- Pull the latest `main` before starting a task.
- Keep commits focused on one task or bug fix at a time.
- Include a short pull request summary of what changed and how it was tested.
- Ask for review before merging into `main`.

## Local Setup

1. Install the .NET 10 SDK.
2. From the repo root, run `dotnet restore`.
3. Start the app with `dotnet run`.
